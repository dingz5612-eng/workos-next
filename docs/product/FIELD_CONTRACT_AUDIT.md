# Field Contract Audit

This audit freezes the current acceptance contract for the two production
sample loops. The goal is to make every card understandable before adding more
backend behavior.

Rules:

- Creation identifiers are typed by the operator or generated by the system.
  They are not search selects.
- Reference fields select an existing projection candidate.
- Bounded business choices use option sets.
- Dates and periods use date or date-time controls.
- System-generated values are read-only and must declare their source.

## Accommodation.CheckIn

This slice is the hostel folio ledger loop. It follows the real business path:
lead -> booking -> resident -> bed assignment -> tariff/folio -> deposit
liability -> front-desk payment -> finance reconciliation -> check-in ->
operating metrics.
