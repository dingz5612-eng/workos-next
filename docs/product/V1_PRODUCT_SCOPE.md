# WorkOSNext V1.0 Product Scope

## Must Have

- Mobile-first experience.
- Chinese and Russian support.
- Intent Hub for active work.
- Work Queue for passive work.
- Object Workspace.
- Task Surface.
- Confirm Sheet.
- Guidance Dock.
- After Action result.
- Accommodation reference domain.
- Maintenance reference domain.
- Behavior event collection placeholder.
- Basic metric placeholder.
- Health API.

## V1.0 Reference Domains

Accommodation:

- Resident.
- Room / bed.
- Accommodation application.
- Stay order.
- Deposit.
- Check-in.
- Fee material.
- Checkout.
- Settlement confirmation.

Automotive repair:

- Vehicle.
- Driver.
- Repair request.
- Technician assignment.
- Diagnosis.
- Repair handling.
- Inspection.
- Parts requirement.
- Fee material.
- Repair order close.

## Not In V1.0

- Full finance ledger.
- Full BI dashboard.
- Model training.
- Automatic high-impact action execution.
- Traditional module menu as primary work model.
- Migration of legacy FunRide pages.

## Success Criteria

- A user can understand the active path: "I want to find or do something."
- A user can understand the passive path: "This is what I should handle today."
- Accommodation and maintenance share the same workspace/card projection contract.
- All user-visible copy supports Chinese and Russian.
- Critical actions show confirmation before completion.
- Behavior data starts accumulating for later personalization.
