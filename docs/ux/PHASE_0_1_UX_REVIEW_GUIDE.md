# Phase 0-1 UX Review Guide

## What To Review

Open the mobile prototype and evaluate:

- Is the home page clear without a module menu?
- Is "search / intent" understandable as the active entry?
- Is "workbench" understandable as the passive entry?
- Does the object page feel centered on the business object?
- Does the task page clearly do only one thing?
- Does the confirm sheet make high-impact actions feel safe?
- Does Chinese/Russian switching cover all visible text?
- Does the help page explain AI boundaries clearly?

## Review Script

1. Open the app.
2. Switch between Chinese and Russian.
3. Use the home search entry.
4. Open a stay order object.
5. Enter the deposit task.
6. Prepare the action.
7. Confirm the action.
8. Return to workbench.
9. Open the maintenance work item.
10. Submit feedback from the task guidance area.

## Feedback Format

Record issues as:

```text
Page:
Language:
What felt wrong:
Expected behavior:
Priority:
```
