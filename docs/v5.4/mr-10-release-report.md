## Rollback

RollbackInstruction:
`docs/v5.4/mr-10-rollback-instruction.json`

Rollback is feature-flag / runtime-mode based:

- `PeriodAnalytics` active/pilot -> disabled
- `RiskCommand` active/pilot -> previous dashboard / limited risks
- `PCGovernance` active/pilot -> feature flags off or limited read-only mode

Forbidden rollback actions:

- delete closed period snapshot
- overwrite frozen snapshot
- delete period late adjustment
- delete export audit
- SQL edit finance snapshot body as source of truth

## Compensation

CompensationInstruction:
`docs/v5.4/mr-10-compensation-instruction.json`

Allowed append-only compensation paths:

- `PeriodLateAdjustmentRecorded`
- Correction note
- Risk item re-generation

Compensation must preserve original closed period snapshots, export audit,
risk source refs, and correction evidence. Bad period output is corrected by a
late adjustment or correction note, not by mutating a frozen snapshot.
