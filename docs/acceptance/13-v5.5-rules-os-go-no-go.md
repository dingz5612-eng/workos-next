# V5.5 Rules OS Go / No-Go Acceptance

V5.5 Rules OS can move to the next batch only when the current batch has durable
evidence and its dependency batches are green.

## A Batch Acceptance

A batch is complete only when:

- Rule authority exists in `docs/engineering/00-rule-authority.md`.
- The Operations Runtime target axis is documented.
- `docs/architecture` is explicitly marked as compatibility reference.
- The PR template requires rule authority and batch dependency checks.
- `scripts/check-rule-authority.mjs` passes locally and through architecture
  guard.

## Final V5.5 Acceptance

Final V5.5 Rules OS Go requires:

- CI green.
- V5.4 Control Plane Guards green.
- No unclassified non-GET `/api` route.
- No business write outside Operations Confirm unless explicitly classified as
  allowed governance provisional evidence.
- Fact ownership registry and writer allowlist checks pass.
- MR contracts validate.
- P0 invariants have executable evidence or an explicit target maturity and
  No-Go mapping.
- GateResult evidence is machine-generated and bound to `ci_run_id`.
- Rule drift reports are generated and have no P0 drift.
- Final Go/No-Go report is machine-generated from current evidence.

If any P0 condition is false, status is `NO-GO` and business batches remain
blocked.
